# 安全策略

## 支持范围

安全修复优先应用于 `main` 分支的最新版本。公开测试阶段不承诺为旧版本维护单独安全补丁分支。

## 报告漏洞

请不要在公开 Issue 中披露尚未修复的漏洞、API Key 或可直接利用的细节。优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私密报告入口；如果该入口尚未启用，请先创建一个不含漏洞细节的普通 Issue，请求维护者提供私下联系渠道。

报告应包含受影响版本、影响范围、复现条件、最小复现步骤和建议缓解方式。请使用测试账号与无权限测试 Key，不要访问、下载或修改他人的数据。

## 敏感信息

项目不会要求贡献者在 Issue 或 Pull Request 中提供真实 API Key。日志与截图必须移除 Authorization Header、完整 Data URL、私人提示词、个人身份信息和敏感原图。
