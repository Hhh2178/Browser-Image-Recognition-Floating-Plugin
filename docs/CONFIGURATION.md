# 模型与权限配置

## 接口兼容要求

扩展发送 OpenAI Chat Completions 风格请求：`messages` 中包含文本内容块，以及使用 `image_url` 的图片内容块。接口必须同时兼容该请求结构并支持视觉输入；只有文本兼容并不足够。

## 服务商字段

| 字段 | 用途 | 常见问题 |
| --- | --- | --- |
| 服务商名称 | 本地显示名称 | 不影响实际请求 |
| API 地址 | Base URL 或完整 Endpoint | 地址必须使用 `http` 或 `https` |
| Endpoint 模式 | 决定是否自动补全路径 | 模式选错通常导致 404 |
| 图片传输 | 决定发送 Data URL、源地址或纯文本 | 服务商与源网站限制不同 |
| API Key | 请求认证 | 只保存在 `chrome.storage.local` |
| 启用 | 是否参与分析路由 | 禁用后其模型不会被选择 |

## Endpoint 模式

### Base URL

适合填写类似 `https://api.example.com/v1` 的地址。末尾为 `/v1` 时扩展追加 `/chat/completions`；其他 Base URL 会追加 `/v1/chat/completions`。

### 完整 Endpoint

适合服务商要求自定义完整路径时使用。扩展会直接使用清理尾部斜杠后的地址，不再补全路径。

如果地址本身已经以 `/chat/completions` 结尾，即使选择 Base URL，扩展也不会重复追加。

## 图片传输

- **自动**：优先使用浏览器已经读取或生成的 Data URL，没有本地数据时再使用源图片地址。建议从此项开始。
- **Data URL**：把图片内容直接放入请求，适合截图和允许浏览器读取的网页图片；可能受到请求体大小限制。
- **源地址**：让模型服务商从公开 URL 获取图片。登录态图片、临时链接、内网地址或防盗链图片通常不可用。
- **仅文本**：不发送图片，用于验证文本接口链路；不能代表视觉连接成功。

## 模型与每日上限

每个服务商可以配置多个模型。模型标识必须与服务商实际接受的 ID 完全一致，并且支持视觉输入。

每日使用上限是扩展本地计数，用于达到设定次数后选择其他已启用模型。它不会读取或限制服务商官方账单，不应当作成本结算依据。更改系统日期、清理扩展数据或在不同设备使用都会影响本地计数含义。

## 连接测试

“测试当前模型”会先保存有效设置、申请接口 Origin 权限，再向当前模型发送一个最小视觉请求。测试成功说明基本认证、路径、模型和视觉输入链路可用，但不能保证所有网页图片、尺寸和模板都成功。

## 团队配置

团队配置包含非敏感模型设置和自定义模板，不包含 API Key。导入后会尽量保留当前电脑上已有的本地 Key。自定义模板正文仍可能包含团队敏感信息，分享前应人工检查。

## 权限说明

- `contextMenus`：注册图片右键入口。
- `storage`：保存设置、Key、模板与偏好。
- `activeTab`：在用户主动操作时访问当前标签页。
- `scripting`：按需注入工作台和选图逻辑。
- `downloads`：导出本地历史结果。
- 模型 API Origin：保存服务商设置时按实际地址申请。
- `http://*/*` 与 `https://*/*` 可选权限：仅在启用悬停分析时请求。

生产 Manifest 默认没有永久 `<all_urls>` 权限。关闭悬停功能后，扩展会撤销动态脚本和相应权限。

## 安全建议

- 使用最小权限、可轮换、可设置额度的 API Key。
- 不要使用生产系统的管理员 Key。
- 不要把 Key 写入提示词模板、团队配置、Issue 或截图。
- 对敏感图片先检查模型服务商的数据保留、训练和地域政策。
- 共用电脑上使用时，退出前删除本地 Key 和敏感历史。

常见连接错误见[故障排查](TROUBLESHOOTING.md)。
