# 故障排查

## 推荐诊断顺序

1. 确认当前是普通 `http` 或 `https` 网页，不是 Chrome 受保护页面。
2. 打开“模型与接口”，保存配置并运行“测试当前模型”。
3. 确认服务商和模型均已启用，当前模型支持图片输入。
4. 检查 API 地址与 Endpoint 模式是否匹配。
5. 将图片传输保持为“自动”，再重试一次。
6. 打开 `chrome://extensions`，刷新扩展和目标网页。
7. 仍无法解决时，收集脱敏信息后创建 Issue。

## 悬浮窗没有出现

- `chrome://`、Chrome 网上应用店和部分内置页面不允许注入。
- 刚安装或刷新扩展后，原来已经打开的网页可能需要刷新。
- 页面图片可能是背景图、Canvas 或特殊组件，可点击扩展图标使用手动选图，或改用截图快捷键。
- 如果页面刚导航完成，等待加载结束后重试。

## 右键后进入手动选图

右键事件没有提供可用图片地址，或目标是无法直接解析的缩略图链接。点击页面上的真实目标图片即可；如果图片不可选，使用截图分析。

## 接口返回 401 或 403

- 检查 API Key 是否有效、是否包含额外空格、是否有调用该模型的权限。
- 确认 API 地址属于该 Key 对应的服务商。
- 重新保存配置，以便 Chrome 申请接口 Origin 权限。
- 不要把完整 Key 或 Authorization Header 发到 Issue。

## 接口返回 404

Base URL 模式会补全 `/chat/completions`；完整 Endpoint 模式不会。常见错误是 API 地址重复包含 `/v1`，或服务商并不兼容 Chat Completions 路径。根据服务商文档修正地址和模式。

## 文本测试成功，但图片分析失败

- 当前模型可能只支持文本，不支持 `image_url` 内容块。
- 服务商可能不接受 Data URL。将图片传输改为“源地址”进行测试；只有公开可访问图片适合此模式。
- 源网站可能阻止浏览器读取图片。尝试截图分析。
- 图片尺寸或编码可能超过服务商限制，请换较小图片验证。

## 一直提示没有可用模型

确认至少有一个已启用的服务商及已启用模型，且配置了 API Key。若设置了每日使用上限，检查是否所有模型都达到本地上限。

## 历史消失或没有同步

历史仅保存在当前浏览器的扩展本地数据库中，不提供账号或云同步。无痕模式、浏览器清理、重装或卸载扩展都可能影响本地数据。重要结果应及时导出。

## 创建 Issue 时请提供

- 扩展版本和最近一次 commit（如从源码构建）。
- Chrome 版本与操作系统。
- 触发入口：右键、工具栏选图、快捷键或悬停按钮。
- 可复现的最小步骤、预期结果和实际结果。
- 服务商类型、Endpoint 模式、模型标识和图片传输方式。
- 已脱敏的错误文本或截图。

请删除 API Key、Authorization Header、完整 Data URL、私人提示词和敏感原图。
