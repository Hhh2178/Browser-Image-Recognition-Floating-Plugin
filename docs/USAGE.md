# 使用手册

## 开始之前

你需要准备一个支持图片输入、并兼容 OpenAI Chat Completions 请求格式的模型接口。扩展不提供账号、API Key 或免费额度。

首次使用建议按以下顺序操作：安装扩展 → 配置模型 → 测试连接 → 打开普通网页 → 选择图片 → 选择模板 → 开始分析。

## 打开工作台的四种方式

### 图片右键菜单

在网页图片上点击右键，选择“Hhh：悬浮窗分析图片”。这是最直接的入口。对于图片外层包裹链接的缩略图，扩展会尝试识别实际图片目标。

### 工具栏手动选图

点击 Chrome 工具栏中的扩展图标。页面进入选图状态后，点击目标图片；按 `Esc` 取消。

### 截图快捷键

按 `Ctrl+Shift+Y` 捕获当前标签页可见区域并打开分析工作台。快捷键可以在 `chrome://extensions/shortcuts` 中修改。

### 可选悬停入口

在“行为与权限”中启用“网页图片悬停分析”。Chrome 会要求网站访问权限；启用后，尺寸足够的图片会显示快捷分析按钮。关闭后扩展会撤销相应的动态脚本和权限。

## 使用悬浮工作台

1. 确认来源摘要中的网页标题和图片类型。
2. 选择提示词模板。
3. 选择中文、英文或 JSON 输出；可选项取决于模板支持范围。
4. 点击“开始分析”。
5. 在任务队列中查看进行中、完成或失败的任务。
6. 复制结果，或从历史记录集中导出。

桌面端可以拖动标题栏移动工作台，也可以调整窗口大小。窄屏页面会自动限制工作台尺寸，避免超出可视区域。

## 提示词模板

内置模板覆盖通用图片反推和网页截图分析。自定义模板可以指定名称、任务类型、描述、标签、正文和支持的输出格式。

模板正文可使用页面与输出相关变量。导入旧版本 JSON 时，扩展会兼容旧的 `prompt` 字段并迁移为当前 `content` 字段；无效条目会报告失败数量，不会静默忽略。

建议让模板明确说明：分析对象、需要提取的维度、输出语言或结构、是否只返回最终结果。不要在模板中长期保存密码、客户隐私或未公开业务数据。

## 多服务商与多模型

每个服务商共享 API 地址、API Key、Endpoint 模式和图片传输方式；其下可以添加多个模型。只有已启用的服务商和模型会参与分析路由。

你可以为模型设置每日使用上限。达到上限后，路由会尝试选择其他可用模型。这里的计数只是扩展本地的调用次数控制，不代表服务商账单或官方配额。

## 历史与导出

扩展仅保留最近 50 条成功结果。历史位于当前浏览器本机，不会自动同步到其他设备。导出功能会下载尚未标记为已导出的结果，成功下载后再标记记录，避免下载失败导致数据被误判为已导出。

删除单条历史或清空历史是本地操作。卸载扩展通常也会清除该扩展的本地存储，请先导出需要保留的内容。

## 团队配置

团队配置适合在成员之间共享非敏感模型设置和自定义模板。导出的 JSON 不包含 API Key；导入时会尽量保留当前电脑上相同服务商 ID 对应的本地 Key。

分享配置前仍应检查文件内容，避免自定义模板本身含有敏感信息。

## 页面限制

Chrome 不允许扩展在部分受保护页面注入脚本，例如 `chrome://` 页面、Chrome 网上应用店和某些浏览器内置页面。网页的跨域策略、懒加载、Canvas 渲染或登录状态也可能导致源图片无法直接读取；遇到此类情况可尝试截图分析。
