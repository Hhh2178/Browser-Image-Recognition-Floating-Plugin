# 应用场景与边界

## 参考图提示词反推

适合提取主体、构图、景别、镜头、光线、色彩、材质、环境与整体风格，再整理为可复用的图片生成提示词。

推荐流程：右键参考图 → 选择“通用图片反推”模板 → 选择目标语言 → 分析 → 人工检查品牌、人物身份和受版权保护元素。

模型输出是解释和推测，不是原始创作者的真实提示词，也不能保证在另一生成模型中完全复现原图。

## 网页与 UI 截图分析

适合评审页面布局、信息层级、组件风格、留白、色彩和视觉重点。可以用于设计复盘、竞品研究、改版前记录与前端实现沟通。

扩展不会读取页面 DOM 结构来生成精确设计稿；截图分析得到的是视觉层面的观察。无障碍、交互逻辑、性能和响应式行为仍需要专门工具验证。

## 广告、电商与内容素材拆解

适合分析商品主图、社交媒体海报、Banner、封面和信息流素材的视觉结构，形成创意摘要或结构化字段。

不要将模型判断当作广告平台数据、转化效果或法律合规结论。人物身份、商标、价格与文字识别应人工核对。

## 模型与提示词对比

同一张图可以使用相同模板在不同视觉模型间对比输出。多服务商配置和本地每日上限可以帮助控制实验范围。

要做可重复评测，应另外记录模型版本、服务商参数、请求时间和人工评分标准；扩展当前不是完整的基准测试平台。

## 团队知识沉淀

团队可以共享不含 API Key 的模型配置与自定义模板，再将成功结果导出到既有知识库。当前没有账号系统或云同步，跨设备同步需要用户自行完成。

## 不建议使用的场景

- 医疗诊断、法律判断、金融决策或其他高风险专业结论。
- 人脸身份确认、年龄或敏感属性推断。
- 处理未经授权的机密图片、个人身份信息或客户隐私。
- 需要像素级测量、精确 OCR、完整 DOM、网络请求或页面交互审计的任务。
- 需要离线处理的任务；模型请求会发送到用户配置的第三方服务商。
- 视频逐帧分析或大规模无人值守批处理；当前版本面向单张图片和可见页面截图。

## 结果质量建议

- 使用明确、可验证、只包含一个主要目标的模板。
- 优先选择主体清晰、分辨率足够的图片。
- 对事实、文字、品牌和人物身份进行人工复核。
- 使用结构化 JSON 时，在下游再次校验字段和数据类型。
- 对敏感素材先确认所选模型服务商的数据保留和训练政策。
