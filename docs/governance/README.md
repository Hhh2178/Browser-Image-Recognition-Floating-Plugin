# Governance

This folder owns durable project rules. It does not hold feature specs, daily notes, or release narratives.

## Documents

- `root-entry-responsibility-matrix.md`: what root files own and must not duplicate.
- `engineering-guardrails.md`: safety, testing, generated artifact, and secret-handling rules.
- `release-version-policy.md`: semantic versioning, tags, GitHub release, and update boundaries.

## Update Trigger

Update governance docs when the project changes how agents work, verify, publish, or protect user data.
