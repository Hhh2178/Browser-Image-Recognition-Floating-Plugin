# 浏览器图片分析插件重构设计

日期：2026-07-15

工作名称：Hhh Prompt Studio

## 1. 背景

现有插件是 Manifest V3 扩展，支持网页图片分析、截图分析、视频拉片、提示词模板、模型配置、Eagle 导出和美术提案等功能。静态审计发现以下主要问题：

- 主入口 `content.js` 超过 1 万行、约 405 KB，UI、状态、媒体处理、请求和导出高度耦合。
- 多个文件存在中文编码损坏风险。
- UI 依赖大型内联 HTML/CSS 和手动 DOM 更新，状态复杂且难以测试。
- 默认声明 `<all_urls>`，权限范围大于核心工作流所需。
- 缺少 TypeScript、构建系统和自动化测试。

本项目在独立目录中重写核心，不直接修改原插件。原插件继续作为行为参考和迁移来源。

## 2. 产品目标

为创意小团队提供稳定、高频、可诊断的网页图片与截图分析工具。首版优先保证：

- 从当前网页快速取得图片或截图。
- 使用可管理的提示词模板调用 OpenAI 兼容视觉模型。
- 清晰呈现中文、英文或 JSON 结果。
- 支持悬浮与右侧停靠两种工作台模式。
- 保存最近分析历史，并提供安全的团队配置导入导出。
- 以模块化、可测试架构支持后续视频拉片和美术提案。

## 3. 首版范围

### 3.1 包含

- 网页图片右键分析。
- 工具栏按钮打开工作台。
- 截图快捷键分析当前可见区域。
- 可选的网页图片悬停分析按钮。
- 悬浮工作台与右侧停靠模式切换。
- OpenAI 兼容接口、自定义 Base URL 或完整 Endpoint。
- 模型预设、连接测试和脱敏诊断。
- 提示词模板选择、新建、复制、编辑、删除、校验、导入和导出。
- 中文、English、JSON 三种结果格式。
- 最近 50 条本地历史，支持搜索、恢复和删除。
- 旧插件模板 JSON 的兼容导入。
- 明暗主题和基本无障碍支持。

### 3.2 不包含

- 视频拉片和关键帧提取。
- 美术提案海报生成。
- Eagle 导出。
- 云同步、账号体系和远程团队模板库。
- 完整素材库、标签收藏和长期保存原始大图。
- Gemini 原生接口或其他非 OpenAI 兼容协议。

这些能力保留为后续版本方向，不在首版中预留未验证的 UI。

## 4. 技术方案

使用 WXT、React 和严格 TypeScript 构建 Manifest V3 扩展。

### 4.1 入口

- `background`：注册右键菜单、工具栏、快捷键、截图、权限和网络请求。
- `content`：在 Shadow Root 中挂载 React 工作台。
- `options`：管理模型、权限、导入导出和高级设置。
- 可选注册的 hover content script：仅在用户启用悬停分析并授予权限后运行。

### 4.2 模块边界

- `media-core`：图片读取、来源校验、尺寸检测、压缩、截图处理。
- `analysis-core`：模板渲染、请求策略、超时、取消、响应解析。
- `prompt-library`：模板 schema、内置模板、自定义模板 CRUD、校验和迁移。
- `settings`：模型、界面偏好、快捷行为和权限状态。
- `history`：成功记录、缩略图、搜索、容量限制和删除。
- `messaging`：后台与内容 UI 之间的类型化消息协议。
- `diagnostics`：错误分类、脱敏和可复制诊断。

各模块通过导出的类型和接口通信，不读取彼此内部状态。核心模块不依赖 React 或 Chrome 全局对象，以便单元测试。

## 5. 数据流

1. 用户通过右键、工具栏、快捷键或可选悬停按钮发起分析。
2. Background 验证当前标签页是否可注入，并发送统一启动消息。
3. Content UI 挂载或恢复工作台，将素材交给 `media-core`。
4. `media-core` 验证类型和尺寸，生成适合请求的图片数据与缩略图。
5. 用户选择模板、模型和输出格式。
6. `prompt-library` 校验并渲染模板；`analysis-core` 生成类型化请求。
7. Background 使用本地 API Key 向已授权 Endpoint 请求，返回标准化成功或错误对象。
8. UI 显示结果；成功任务写入历史，失败任务不占用历史配额。

API Key、Authorization Header 和完整图片 Data URL 不进入网页上下文、日志、诊断或导出文件。

## 6. 工作台设计

### 6.1 模式

- 默认打开为可拖动悬浮窗。
- 用户可通过标题栏吸附图标切换为右侧停靠。
- 两种模式共享同一个 React 组件树和任务状态。
- 记住每位用户最后的模式、悬浮位置和尺寸。
- 窄窗口使用全宽覆盖回退，避免挤压或破坏网页布局。

### 6.2 信息层级

工作台从上到下固定为：

1. 素材预览与来源信息。
2. 提示词模板和模型选择。
3. 中文、English、JSON 输出格式。
4. 主分析操作与运行状态。
5. 结果、复制、重试和诊断。

历史使用抽屉；复杂模型配置和导入导出进入 Options 页面。工作台不使用多步向导，重复分析时无需切换页面。

中文、English、JSON 标签同时承担目标格式选择和结果切换。每次分析请求只生成当前选中的一种格式，成功结果按格式缓存；切换到尚未生成的格式时显示明确的“生成此格式”操作，不自动产生额外请求。历史中的每条记录只保存该次请求的目标格式和对应结果。

### 6.3 入口默认值

- 默认开启：右键图片、工具栏、截图快捷键。
- 默认关闭：悬停分析按钮。
- 用户启用悬停功能时才申请全站可选权限。

## 7. 提示词模板库

模板是首版核心领域对象。

### 7.1 模板字段

- `id`：稳定唯一标识。
- `name`：用户可见名称。
- `taskType`：`image_analysis` 或 `screenshot_analysis`。
- `description`：工作台中的简短用途说明。
- `tags`：筛选标签。
- `content`：模板正文。
- `supportedFormats`：允许的输出格式集合。
- `source`：`builtin` 或 `custom`。
- `schemaVersion`：迁移版本。
- `createdAt`、`updatedAt`：本地时间戳。

首版支持受控变量，例如 `{{outputFormat}}`、`{{pageTitle}}` 和 `{{sourceType}}`。未知变量、空内容、重复 ID 和不支持的输出格式会阻止保存并显示具体错误。

### 7.2 操作

- 工作台快速选择和按任务类型筛选。
- 新建自定义模板。
- 复制任意模板。
- 编辑和删除自定义模板。
- 内置模板只读，可复制后修改。
- JSON 导入导出。
- 导入冲突时明确选择覆盖或另存，不静默覆盖。
- 旧插件模板导入失败时逐条报告原因。

## 8. 模型与请求

首版只实现 OpenAI Chat Completions 兼容协议。

- 支持 Base URL 自动补全和完整 Endpoint。
- 支持自定义模型名。
- 支持 `auto`、`data_url`、`source_url` 和 `text_only` 图片传输策略。
- 提供文本连接测试和视觉连接测试。
- 支持超时、取消和可重试判断。
- 不在 CI 或测试夹具中使用真实 API Key。

团队配置导出包含 Endpoint、模型和非敏感偏好，不包含 API Key。每位成员在本机重新输入密钥。

## 9. 历史

- 使用 IndexedDB 保存最多 50 条成功记录。
- 每条包含缩略图、来源元数据、模板 ID、模型 ID、输出格式、结果和时间戳。
- 不长期保存完整原始图片。
- 超过容量时按最旧记录自动淘汰。
- 支持文本搜索、重新打开、单条删除和清空全部。
- 恢复历史不会自动重新发送模型请求。
- 删除和清空操作需要明确确认。

## 10. 状态与错误

错误对象至少包含 `code`、`stage`、`message`、`retryable`、`suggestedAction` 和脱敏 `diagnostic`。

必须区分：

- 配置缺失。
- 当前页面不可注入。
- 权限被拒绝。
- 图片读取、解码、类型或尺寸错误。
- API Origin 未授权。
- 网络连接失败。
- 请求超时或用户取消。
- HTTP 错误。
- 无效 JSON。
- 空响应。
- 模型不支持视觉输入。
- 模板变量或返回结构不合法。

错误在结果区原位展示，保留当前素材和配置，并提供合适的设置、重试或复制诊断操作。

诊断可以包含错误代码、阶段、耗时、模型名、Endpoint 域名、图片数量和大小。诊断不得包含密钥、Authorization Header、完整 Data URL、完整 Prompt 或未脱敏原始响应。

## 11. 权限

默认权限：

- `contextMenus`
- `storage`
- `activeTab`
- `scripting`
- `commands`

不默认声明 `<all_urls>`。

- 右键、工具栏和快捷键依赖用户手势产生的临时 `activeTab` 权限。
- 启用悬停分析时，通过 `optional_host_permissions` 请求网页访问权限。
- 保存或测试模型 Endpoint 时，只请求对应 API Origin。
- Chrome 内部页面和其他禁止注入的页面始终显示明确说明。

## 12. 视觉与交互规范

- 面向创意团队的紧凑工具界面，不采用营销页构图。
- 中性灰为主，单一蓝色作为操作强调色，成功、警告和错误使用语义色。
- 圆角不超过 8px，依靠间距和边框建立层级，避免卡片嵌套。
- 图标使用 Lucide，陌生图标带 tooltip。
- 固定工具栏、按钮和结果区域的稳定尺寸，状态变化不引起布局跳动。
- 支持键盘焦点、可见 focus ring、ARIA 标签和 `prefers-reduced-motion`。
- 明暗主题使用同一套语义令牌。
- 首版动画只用于模式切换、抽屉和状态反馈。

## 13. 测试策略

### 13.1 单元测试

- API URL 和 Endpoint 规范化。
- 图片传输策略和请求构建。
- 模板 schema、变量替换、导入冲突和旧格式迁移。
- 响应解析和错误分类。
- 超时、取消和重试判断。
- 诊断脱敏。
- 历史容量、排序和淘汰。

### 13.2 组件测试

- 悬浮和停靠切换不丢失状态。
- 模板选择与 CRUD。
- 模型配置和连接状态。
- 中文、English、JSON 标签切换。
- 加载、取消、错误、空状态和成功状态。
- 历史搜索、恢复和删除。

### 13.3 集成与 E2E

使用固定测试网页、测试图片和本地 Mock API：

- 右键图片打开工作台。
- 工具栏和截图快捷键打开正确任务。
- 素材读取、压缩、请求、解析和历史入库完整链路。
- 模板导入导出和旧格式迁移。
- 可选权限启用与关闭。
- 刷新后恢复模式、尺寸和用户偏好。
- 1366 × 768、1920 × 1080 和窄窗口无重叠、溢出或不可达操作。

完成前必须通过 TypeScript strict、ESLint、格式检查、Vitest、Chrome E2E 和 production build。

## 14. 迁移与交付

- 新项目使用独立 Git 仓库，不修改原插件目录。
- 通过适配器导入原插件导出的模板 JSON。
- 不尝试跨扩展 ID 自动读取旧 `chrome.storage`。
- 模型密钥必须在新插件中重新输入。
- 构建产物可直接作为 Chrome unpacked extension 加载。
- 提供安装、配置、开发、测试和打包文档。

## 15. 验收标准

首版在以下条件全部满足时完成：

- 四种入口按默认和可选策略工作。
- 悬浮与停靠模式可无损切换并持久化。
- 图片与截图可通过 Mock API 完成分析。
- 三种结果格式可正确呈现和复制。
- 模板库 CRUD、校验、导入导出和旧格式迁移通过。
- 模型连接测试、错误分类和脱敏诊断通过。
- 最近 50 条历史按规则保存、搜索、恢复和删除。
- 默认不请求 `<all_urls>`。
- 自动化测试和 production build 全部通过。
- 目标窗口尺寸下完成视觉和交互检查。
